# 補助金書類作成 API自動化設計

## ゴール

「Googleドライブに資料を入れたら、申請書類が自動で出来上がる」状態を作る。

---

## 全体の流れ

```
顧客から資料が届く
      |
      v
Google Driveの顧客フォルダに格納
      |
      v
Google Sheetsの管理画面で「実行」ボタンを押す
      |
      v
Cloud Functions（Python）が起動
      |
      +---> Google Drive APIで資料をダウンロード
      +---> Claude API（Sonnet）でPDF読取・データ抽出
      +---> openpyxlでExcel生成
      +---> Google Drive APIで完成ファイルをアップロード
      |
      v
Google Sheetsに「完了」と表示 + Driveに出力ファイル
```

---

## 具体的な構成

### 1. Google Drive（資料の置き場所）

```
共有ドライブ/
├── 補助金申請/
│   ├── テンプレート/
│   │   ├── 【原本】通常枠_法人2026.xlsx
│   │   └── 【原本】インボイス枠_法人2026.xlsx
│   │
│   ├── 顧客データ/
│   │   ├── 京のお肉処弘/
│   │   │   ├── 入力/          ← ここに資料を入れるだけ
│   │   │   │   ├── ヒアリングシート.xlsx
│   │   │   │   ├── 履歴事項全部証明書.pdf
│   │   │   │   ├── 損益計算書.pdf
│   │   │   │   └── 給与データ/
│   │   │   │       ├── 3月度.pdf
│   │   │   │       └── ...
│   │   │   │
│   │   │   └── 出力/          ← ここに完成ファイルが出る
│   │   │       ├── 京のお肉処弘_通常枠_AI版.xlsx
│   │   │       └── 京のお肉処弘_給与支給総額計算.xlsx
│   │   │
│   │   ├── オーサワジャパン/
│   │   │   ├── 入力/
│   │   │   └── 出力/
│   │   └── ...
```

**メンバーの作業: フォルダを作って、資料を「入力」に入れるだけ**

### 2. Google Sheets（管理画面）

| 顧客名 | 申請枠 | DriveフォルダID | 状態 | 実行 |
|--------|--------|---------------|------|------|
| 京のお肉処弘 | 通常枠 | 1A2B3C... | 完了 | [実行] |
| オーサワジャパン | インボイス枠 | 4D5E6F... | 未処理 | [実行] |
| 新規顧客 | 通常枠 | 7G8H9I... | 処理中... | - |

- 顧客名と枠を入力 → 「実行」ボタンを押す → 完了まで待つ
- ステータスが「完了」になったらDriveの出力フォルダを確認

### 3. バックエンド（Cloud Functions / Cloud Run）

```
Python処理の流れ:

1. Google Drive APIでフォルダ内のファイル一覧取得
2. PDF/Excelをダウンロード
3. PDFを画像に変換（PyMuPDF）
4. Claude Sonnet APIに画像を送信、構造化データを取得:
   - 履歴事項 → 法人情報JSON
   - 損益計算書 → 財務データJSON
   - 給与一覧 → 従業員別データJSON
   - ヒアリングシート → 申請内容JSON（openpyxlで直接読む方が正確）
5. テンプレートExcelに転記（openpyxl）
6. 出力ファイルをDriveにアップロード
7. Sheetsのステータスを更新
```

## 開発ステップ

### Phase 1: コア処理（1-2日）
最優先。ローカルPCで動く状態を作る。

```python
# transfer_api.py のイメージ
def process_company(folder_path, template_type):
    """1社分の処理"""
    # 1. ファイル一覧取得
    files = scan_folder(folder_path)

    # 2. ヒアリングシート読取（openpyxlで直接）
    hearing = read_hearing_sheet(files['hearing'])

    # 3. PDF読取（Claude API）
    registry = extract_from_pdf(files['registry'], PROMPT_REGISTRY)
    tax = extract_from_pdf(files['tax'], PROMPT_TAX)
    pl = extract_from_pdf(files['pl'], PROMPT_PL)

    # 4. テンプレートに転記
    output = fill_template(template_type, hearing, registry, tax, pl)

    return output
```

### Phase 2: Google Drive連携（1日）
Drive APIでファイルのダウンロード/アップロードを追加。

### Phase 3: Google Sheets管理画面（1日）
Sheetsのボタンからバックエンドを呼び出す連携。

### Phase 4: Cloud Functionsデプロイ（1日）
ローカル→クラウドに移行。どこからでも実行可能に。

## Claude APIへのプロンプト設計

### 履歴事項全部証明書の読取

```
この履歴事項全部証明書の画像から、以下の情報をJSON形式で抽出してください。
読み取れない場合はnullにしてください。

{
  "法人名": "",
  "法人名フリガナ": "",
  "本店所在地": "",
  "郵便番号": "",
  "設立年月日": "yyyy-mm-dd",
  "資本金": 0,
  "代表者氏名": "",
  "代表者役職": "",
  "事業目的": ["", ""]
}
```

### 損益計算書の読取

```
この損益計算書の画像から、以下の数値をJSON形式で抽出してください。
該当項目がない場合はnullにしてください。金額は円単位の整数で。

{
  "事業年度_開始": "yyyy-mm-dd",
  "事業年度_終了": "yyyy-mm-dd",
  "売上高": 0,
  "売上原価": 0,
  "売上総利益": 0,
  "営業利益": 0,
  "経常利益": 0,
  "当期純利益": 0,
  "販管費": {
    "給料手当": 0,
    "雑給": 0,
    "賞与": 0,
    "役員報酬": null,
    "法定福利費": 0,
    "福利厚生費": 0,
    "減価償却費": 0,
    "旅費交通費": 0
  }
}
```

### 給与支給控除一覧の読取

```
この給与支給控除一覧表の画像から、従業員ごとのデータをJSON配列で抽出してください。

[
  {
    "氏名": "",
    "所属": "",
    "雇用形態": "正社員 or パート・アルバイト",
    "出勤日数": 0,
    "所定労働時間": 0,
    "基本給": 0,
    "課税支給合計": 0,
    "支給合計": 0,
    "控除合計": 0,
    "差引支給額": 0
  }
]

注意:
- 所属欄に「正社員」「アルバイト」の記載があればそれを使う
- 社員番号が100xxx台は正社員、200xxx台はアルバイトの可能性が高い
- 全ページの全従業員を漏れなく抽出する
```
