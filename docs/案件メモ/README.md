# 案件メモ

個別案件で発生した不具合・調査・修正のログを残す場所。
「なぜこの修正を入れたか」「どの案件で何が起きたか」を後から追えるようにする。

---

## ファイル命名

```
YYYY-MM-DD_担当者_会社名.md
```

- 日付は**起票日**（村上さんや顧客から連絡を受けた日）
- 担当者は社内でその案件を動かしている人（複数なら `_` 区切り）
- 会社名は顧客企業名（略称可）

例:
- `2026-04-20_黒石_ピスケス.md`
- `2026-03-10_坂平_ファンファーレ.md`

---

## テンプレート

新しい案件メモを作るときは以下をコピーして使う。

```markdown
# {会社名} 案件メモ

- **起票日**: YYYY-MM-DD
- **起票者**: （誰から連絡を受けたか）
- **社内担当**: （村上さん／坂平さん等）
- **顧客担当**: （黒石さん等）
- **案件種別**: （通常枠法人／インボイス枠 等）

## 1. 報告された症状

- 症状①
- 症状②

## 2. 再現環境・資料

- 使用資料: （ヒアリングシート、賃金台帳など）
- 実行環境: Streamlit / CLI / 手動
- 出力ファイル: （あれば）

## 3. 調査結果

### 症状①
- 原因箇所: [file:line](path/to/file#Lnn)
- 根本原因:
- 影響範囲:

### 症状②
（同上）

## 4. 対策

- [ ] 対策案①
- [ ] 対策案②

採用: 〇〇

## 5. 修正内容

- コミット:
- 変更ファイル:
- テスト結果:

## 6. 備考・再発防止

- （運用で気をつけること、類似案件への示唆など）
```

---

## ステータス運用

ファイル冒頭に以下のいずれかを書いておくと追いやすい。

- **調査中** — 原因特定前
- **対策検討中** — 原因わかったが修正方針未決
- **修正中** — 実装中
- **修正済** — コミット済（コミットハッシュを記載）
- **クローズ** — 顧客確認まで完了
