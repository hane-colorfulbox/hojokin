## 3. 調査結果

### 症状①「ヒアリングシートが読み込めてない」→ 実体は別

ヒアリング自体は検出済み（Streamlitのデバッグ表示に `hearing: ヒアリングシート2026_通常枠法人.xlsx` が出ている）。

真の原因は**症状②の手前で pipeline 全体が例外終了**しており、その結果ヒアリング転記結果も出力されなかったため、村上さんには「ヒアリングが読めていない」ように見えた。

#### 根本原因

[hojokin/pipeline.py:68-71](../../hojokin/pipeline.py#L68-L71) の `FileDetector.get()`:

```python
def get(self, category: str) -> Path | None:
    files = self.files.get(category, [])
    return files[0] if files else None
```

- 拡張子フィルタなし。ファイル名に「賃金台帳」を含むものを全部拾う
- 今回 `.csv` と `.xlsx` の両方が賃金台帳として検出され、`files[0]` が `.csv` になった
- [hojokin/wage_reader.py:520](../../hojokin/wage_reader.py#L520) の `openpyxl.load_workbook(.csv)` が例外を投げ、申請書作成処理が全滅
- ヒアリング転記も実行されずに終了 → 「ヒアリングが読めてない」という症状に見える

#### 影響範囲

- `wage_ledger` カテゴリだけでなく、他カテゴリも同じ `get()` を使っているので、将来同様の拡張子混在で再発しうる
- 特に `estimate`（お見積）、`pl`（損益計算書）あたりも同名で複数拡張子が混ざる可能性
